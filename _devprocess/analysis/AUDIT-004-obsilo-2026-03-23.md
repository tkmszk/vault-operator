### H-1: Prompt Injection bei aktivierter Voll-Auto-Approval (CWE-77)

| Field | Value |
|-------|-------|
| **Severity** | High |
| **CWE** | CWE-77 |
| **Location** | `src/core/tool-execution/ToolExecutionPipeline.ts`, `src/core/prompts/sections/securityBoundary.ts` |
| **Status** | Confirmed -- by design, UI-Warning empfohlen |
| **Previous** | H-1 seit AUDIT-001 -- Unchanged |

**Finding:**
Bei Auto-Approval "permissive" (alle Writes auto-approved) koennte prompt-injizierter Inhalt (z.B. aus web_fetch oder Vault-Notiz) unkontrolliert Vault-Aenderungen ausloesen.

**Vorhandene Mitigations:**
- Mode-Tool-Filter schraenken verfuegbare Tools ein
- System-Prompt-Boundary separiert User-Input von Instructions (`securityBoundary.ts:8-18`)
- IgnoreService/ProtectedService blockieren sensitive Pfade
- Checkpoint-System ermoeglicht Rollback
- Consecutive-Mistake-Limit (default: 3) stoppt Endlosschleifen
- configDir-Write-Block in SandboxBridge
- Default: Auto-Approval ist AUS (`settings.ts:753-771`)

**Remediation:**
- "permissive" als High-Risk kennzeichnen in UI (Warning-Modal bei Aktivierung)
- Enterprise-Preset: auto-approval fuer Writes IMMER deaktiviert

### M-5: ReDoS in Skill-Trigger-Matching ohne safeRegex() (CWE-1333) -- NEW

| Field | Value |
|-------|-------|
| **Severity** | Medium |
| **CWE** | CWE-1333 |
| **Location** | `src/core/context/SkillsManager.ts:112`, `src/ui/AgentSidebarView.ts:829` |
| **Status** | **Resolved** -- safeRegex() implementiert |
| **Previous** | NEW in AUDIT-004 -- Fixed same day |

**Finding:**
Beide Stellen konstruierten `new RegExp(s.trigger, 'i')` aus User-Skill-Frontmatter (YAML trigger-Feld) ohne safeRegex()-Wrapper. In `SelfAuthoredSkillLoader.ts:660` wird safeRegex() korrekt verwendet, aber die Matching-Stellen im SkillsManager und AgentSidebarView nutzen die unsichere Variante.

Ein malicious Trigger-Pattern wie `(a+)+b` in einer SKILL.md koennte katastrophales Backtracking ausloesen und die UI einfrieren. Der Trigger wird bei JEDER User-Nachricht evaluiert.

**Betroffener Code:**

SkillsManager.ts:112:
```typescript
const regex = new RegExp(s.trigger, 'i');
if (regex.test(msgLower)) return true;
```

AgentSidebarView.ts:829:
```typescript
try { if (new RegExp(s.trigger, 'i').test(msgLower)) { matched.add(s.name); continue; } } catch { /* skip */ }
```

**Risk:** UI-Freeze bei jeder Nachricht wenn ein Skill mit malicious Trigger existiert. LLM kann Skills via manage_skill erstellen (erfordert Approval).

**Remediation:**
```typescript
// Ersetze new RegExp(s.trigger, 'i') durch:
import { safeRegex } from '../skills/SelfAuthoredSkillLoader';
const regex = safeRegex(s.trigger, 'i');
if (regex?.test(msgLower)) return true;
```

**Aufwand:** S (2 Zeilen aendern, safeRegex Import hinzufuegen)

### M-6: JSON.parse ohne Schema-Validierung in PPTX-Tools (CWE-502) -- NEW

| Field | Value |
|-------|-------|
| **Severity** | Medium |
| **CWE** | CWE-502 |
| **Location** | `src/core/tools/vault/PlanPresentationTool.ts:210`, `src/core/tools/vault/IngestTemplateTool.ts:767` |
| **Status** | **Resolved** -- Runtime Type Guards implementiert |
| **Previous** | NEW in AUDIT-004 -- Fixed same day |

**Finding:**
Beide Tools parsten LLM-Responses mit `JSON.parse()` und casteten direkt via `as DeckPlan` / `as Record<...>` ohne Runtime-Schema-Validierung.

PlanPresentationTool.ts:210:
```typescript
return JSON.parse(cleaned) as DeckPlan;
```
Mitigiert durch `validatePlan()` (Zeile 223ff) -- prueft slides-Array, required shapes, placeholders, groups.

IngestTemplateTool.ts:767:
```typescript
const enrichments = JSON.parse(cleaned) as Record<string, { ... }>;
```
Mitigiert durch defensive Post-Verarbeitung -- `if (!e) continue;`, optional chaining auf shape_hints.

**Risk:** Unerwartete LLM-Responses koennten zu Runtime-Fehlern fuehren. Kein direktes Security-Risiko (kein Code-Execution), aber Robustheitsproblem.

**Remediation:**
Type Guards vor der Verarbeitung:
```typescript
function isDeckPlan(obj: unknown): obj is DeckPlan {
    const p = obj as Record<string, unknown>;
    return Array.isArray(p?.slides);
}
```

**Aufwand:** S (Type Guard Funktionen hinzufuegen)
