### H-1: Prompt Injection bei aktivierter Voll-Auto-Approval (CWE-77)

| Field | Value |
|-------|-------|
| **Severity** | High |
| **CWE** | CWE-77 |
| **Location** | `src/core/tool-execution/ToolExecutionPipeline.ts`, `src/core/prompts/systemPrompt.ts` |
| **Status** | Confirmed -- by design, Dokumentation empfohlen |
| **Previous** | H-1 seit AUDIT-001 -- Unchanged |

**Finding:**
Bei Auto-Approval "permissive" (alle Writes auto-approved) koennte prompt-injizierter Inhalt (z.B. aus web_fetch oder Vault-Notiz) unkontrolliert Vault-Aenderungen ausloesen.

**Vorhandene Mitigations:**
- Mode-Tool-Filter schraenken verfuegbare Tools ein
- System-Prompt-Boundary separiert User-Input von Instructions
- IgnoreService/ProtectedService blockieren sensitive Pfade
- Checkpoint-System ermoeglicht Rollback
- Consecutive-Mistake-Limit stoppt Endlosschleifen
- configDir-Write-Block in SandboxBridge

**Risk:**
Unkontrollierte Vault-Modifikation bei auto-approval: permissive.

**Remediation:**
- "permissive" als High-Risk kennzeichnen in UI
- Enterprise-Preset: auto-approval fuer Writes IMMER deaktiviert

### M-1: npm-Packages in Sandbox ohne Integritaetspruefung (CWE-494)

| Field | Value |
|-------|-------|
| **Severity** | Medium |
| **CWE** | CWE-494 |
| **Location** | `src/core/sandbox/EsbuildWasmManager.ts:330-400` |
| **Status** | Confirmed |
| **Previous** | M-2 seit AUDIT-002 -- Unchanged |

**Finding:**
esbuild-wasm hat SHA-256-Integrity. Aber `ensurePackage()` laedt npm-Pakete von esm.sh/cdn.jsdelivr.net ohne Integritaetspruefung. `resolveInternalImports()` laedt rekursiv Sub-Dependencies (bis Tiefe 5).

**Risk:**
CDN-Compromise koennte boeswilligen Code in Sandbox einschleusen. Sandbox-Bridge mitigiert Auswirkungen (URL-Allowlist, Rate-Limiting, Path-Validation).

**Remediation:**
- Kurzfristig: User-Warnung beim ersten Package-Download
- Mittelfristig: Known-good-Hashes fuer populaere Pakete
- Langfristig: npm Registry API fuer SRI-Hashes
