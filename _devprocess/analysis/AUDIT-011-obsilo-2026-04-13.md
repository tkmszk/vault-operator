## Findings

### P3: Consider (Info)

#### I-1: execute_vault_op callable without AGENT_INTERNAL_TOOLS filter (CWE-284)

| Field | Value |
|-------|-------|
| **Severity** | Info |
| **CWE** | CWE-284 |
| **Location** | `src/mcp/tools/executeVaultOp.ts:29` |
| **Status** | By Design |

**Finding:** `executeVaultOp` looks up any tool by name from the registry without
checking against `AGENT_INTERNAL_TOOLS`. An MCP client that guesses an internal
tool name (e.g. `switch_mode`, `manage_mcp_server`) could call it.

**Risk:** Low. Internal tools require specific context (active AgentTask, callbacks)
that MCP calls don't provide. Most would fail with a runtime error. The filtering
is on the discovery side (tool list + get_context), not execution.

**Remediation:** Could add an execution-time check, but current approach is acceptable
since internal tools fail gracefully without the right execution context.
