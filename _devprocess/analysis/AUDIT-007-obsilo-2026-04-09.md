### P1: Must Fix (Critical + High)

#### H-1: Unvollstaendige Token-Verschluesselung in GlobalSettingsService (CWE-312)

| Field | Value |
|-------|-------|
| **Severity** | High |
| **CWE** | CWE-312 (Sensitive Data Exposure) |
| **Location** | `src/core/storage/GlobalSettingsService.ts:122-158` |
| **Status** | Resolved |

**Finding:** `GlobalSettingsService.encryptGlobal()` verschluesselte nur `activeModels` und `webTools`, aber nicht die Token-Felder `githubCopilotToken`, `githubCopilotAccessToken`, `kiloToken`, `cloudflareApiToken`, `relayToken`, `mcpServerToken`.

## Positive Findings

Die Codebase zeigt eine reife Sicherheitsarchitektur:

- **XSS-Praevention:** 0 innerHTML-Nutzungen. Konsequent Obsidian DOM API (createEl, createDiv, textContent).
- **Sandbox-Isolation:** Dreistufig -- AstValidator (22 blockierte Patterns) -> vm.createContext (frozen globals) -> OS-Prozess-Isolation. Comment-Stripping verhindert Pattern-Evasion.
- **Path-Traversal-Schutz:** GlobalFileService und recipeValidator mit 3-Layer-Checks (../rejection, absolute path rejection, startsWith-confinement).
- **SSRF-Schutz:** WebFetchTool mit 2-Phasen-Check (Hostname + DNS-Resolution gegen Private IPs).
- **Tool-Governance:** Granulares Approval-System mit Auto-Approval per Kategorie. Self-Modification (manage_source, manage_skill) IMMER manuell. autoApproval-Aenderung durch LLM blockiert.
- **Excessive Agency Controls:** consecutiveMistakeLimit (3), maxIterations (25, soft limit bei 60%), rateLimitMs, maxSubtaskDepth (2), ToolRepetitionDetector (exact + fuzzy).
- **Credential-Handling:** SafeStorageService delegiert an OS-Keychain. Bearer-Token-Auth (nie in URL). Constant-Time SHA-256 Token-Vergleich (Relay). OperationLogger redaktiert sensible Felder.
- **Error-Sanitization:** MCP-Bridge strippt Stack-Traces an API-Grenzen. Keine internen Pfade in User-Fehlermeldungen.
- **Zero Hardcoded Paths:** 34 Stellen korrekt mit `vault.configDir`. Keine hardcodierten `.obsidian`-Strings.
- **Floating Promises:** Alle kontrolliert mit `void` Prefix und `.catch()` Handler.
- **Audit-Annotations:** Konsistente CWE/AUDIT/M-/H-Marker im Code (AUDIT-005, AUDIT-006, M-3, H-5, etc.).
