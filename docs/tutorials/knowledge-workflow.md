# Knowledge workflow overview

The knowledge workflow was originally one long tutorial. In v2.12 it was split into three focused tutorials, because most readers wanted just one piece at a time. This page exists so the old link still leads somewhere useful.

## The three tutorials, in order

1. **[Search your vault by meaning](./search-by-meaning).** Set up the embedding model, build the semantic index, run your first meaning-based search. Twenty minutes (plus indexing time for large vaults).
2. **[Capture a PDF with /ingest](./quick-ingest).** Drop a PDF, get a clean source note with page-level links back to the original. Three minutes per source.
3. **[Sense-making with /ingest-deep](./deep-ingest).** Walk through the guided seven-step dialog that turns a research paper into sense-making notes with paragraph-level provenance. Five to fifteen minutes per source.

Do them in order the first time. After that, the quick path and the deep path are independent.
