# Vault Operator v2.14: Smarter retrieval, controllable reasoning

v2.14 is a feature wave. It sharpens how the plugin finds notes (the Retrieval Wave 1 work, inspired by a close read of Gbrain's pipeline) and adds a per-conversation control surface for how hard the model thinks: a binary thinking toggle and a model-native reasoning-effort slider that reach every default provider, including Amazon Bedrock. Alongside the two headline areas it carries a batch of provider and routing fixes that came out of community issues and live testing.

## v2.14.1 -- adaptive Claude on Bedrock + composer polish

A focused patch on top of v2.14.0 that fixes the reasoning-effort path on Amazon Bedrock for the adaptive Claude lineup and cleans up the chat composer row.

- **Adaptive Claude on Bedrock (Opus 4.7/4.8, Fable, Mythos).** Pinning one of these models with thinking on and a reasoning-effort level other than Auto returned a 400 from Bedrock: `thinking.enabled.budget_tokens: Field required`. The earlier wire shape (`reasoning_config { type: 'enabled', effort: <level> }`) was partially translated by Bedrock into the legacy thinking shape, which then required a `budget_tokens` value those models do not accept. Bedrock now ships the Anthropic-native pair through `additionalModelRequestFields` instead -- `thinking: { type: 'adaptive' }` plus `output_config: { effort }` -- matching the direct Anthropic provider exactly. The budget-token path (Sonnet 4.6, Opus 4.6 and older) is unchanged.
- **GitHub Copilot adaptive thinking shape.** Copilot routes that fall on the adaptive Claude lineup now send `thinking: { type: 'adaptive' }` instead of an unconditional `enabled + budget_tokens`, closing the same class of 400 there. Older Claude on Copilot keeps the legacy shape.
- **OpenRouter adaptive Claude.** When you pin an adaptive Claude on OpenRouter with thinking on, `reasoning.max_tokens` is no longer sent (those models reject `budget_tokens` at the Anthropic layer). The reasoning object stays empty or carries only `effort` if you set one.
- **Composer row cleanup.** The "Thinking On / Thinking Off" chip is gone from the chat composer; the thinking state lives in the picker popover (and in the hover tooltip when forced off). The model name shows the bare core id (e.g. `claude-opus-4-8`) instead of a long provider-supplied display name like `EU Anthropic Claude Opus 4.8 [Cross-Region Profile]` so the rest of the toolbar (attach, more, send) stays in view.
- **Reasoning-effort slider visuals.** The effort knob is now slightly larger than the track, stays fully visible at both ends (`Auto` flush-left, `Max` flush-right) and no longer collapses to a single line. Pointer hit-testing is in sync with the new knob size.

## Highlights

### Retrieval Wave 1

The hybrid search pipeline (semantic + lexical + graph, fused with reciprocal rank fusion) got six focused improvements:

- **Opener-chunk excerpts.** Search results now prefer the note's opening chunk for the displayed excerpt instead of whatever chunk happened to rank first. The opener is almost always the most descriptive line, so result lists read like a table of contents rather than a pile of mid-paragraph fragments.
- **Tokenizer folds umlauts and keeps short acronyms.** The lexical arm now folds `ä/ö/ü/ß` so a query for "Ressourcenübersicht" matches "Ressourcenubersicht" and vice versa, and it stops dropping short all-caps tokens, so "KI", "RAG", and "MCP" survive tokenization instead of being filtered as noise.
- **Weighted RRF with a cosine blend.** Fusion now weights the three arms and blends in the raw cosine similarity as a bonus on top of the rank-based score (bonus-only, so a weak semantic hit can never outrank a strong lexical one). Tag-only matches are down-weighted so a note does not float to the top purely because it shares a tag with the query.
- **Typed graph labels from frontmatter.** Graph-arm hits now carry a typed relationship label derived from the frontmatter property that linked them, so the result explains *why* a note is related, not just that it is.
- **Batched reranking with backoff re-arm.** The cross-encoder reranker now processes candidates in batches and re-arms itself after a transient failure instead of staying disabled for the rest of the session.
- **Reranker ONNX runtime fixed.** The reranker model loads again: it now injects the `onnxruntime-web/wasm` backend with `device: 'auto'` and single-threaded WASM (the bundled Electron runtime has no worker-script URL, which broke the threaded build). Confirmed loading in under two seconds.

Two supporting pieces ship with the wave: near-empty stub notes (under 40 characters of real content) are gated out of the semantic index so they stop ranking on title alone, and a light retrieval bench harness landed for regression-checking fusion changes.

### Per-conversation thinking and reasoning effort

The chat model picker gains two controls that apply to the current conversation without touching your saved model settings:

- **Thinking toggle.** A binary On/Off switch. On models that do not support extended thinking it is silently ignored. The state is mirrored on the chat header so you can see it at a glance.
- **Reasoning-effort slider.** A pill slider with the model's own native effort levels. Claude exposes Low / Medium / High / XHigh / Max; GPT-5 and the o-series expose Minimal / Low / Medium / High. The slider is shown only when you pin a specific model (in Auto mode the tier router already picks the model for the task, so effort stays at the model's own default and no effort field is sent). The leftmost stop is always "Auto", which sends nothing and lets the vendor default apply.

Effort reaches every default provider through its native parameter: Anthropic `output_config.effort`, OpenAI and Copilot `reasoning_effort`, OpenRouter `reasoning.effort`, and Amazon Bedrock `reasoning_config`. Each provider validates the level against the model family, so a GPT-only level can never reach a Claude model and vice versa.

### Amazon Bedrock extended thinking

Bedrock now wires extended thinking for the budget-token Claude lineup (Sonnet 4.6, Opus 4.6 and older). Turning thinking on for one of those models sends `reasoning_config { type: 'enabled', budget_tokens: N }` and forces the request temperature to 1, matching the direct Anthropic provider. The effort-capable lineup (Opus 4.7/4.8, Fable) keeps using the effort enum. A capability fix underpins this: only the adaptive-thinking Claude models accept an effort enum, so the budget-token models no longer offer or send one (they previously triggered a Bedrock 400, "thinking.enabled.budget_tokens: Field required").

## Provider and routing fixes

- **ChatGPT (Codex OAuth) connects again.** The Codex backend gates models on the client version it is told; the plugin now reports the current Codex CLI version and discovers the real model list (gpt-5.5 / gpt-5.4 / gpt-5.4-mini) instead of a stale hard-coded set. The OAuth flow also reads the nested `chatgpt_account_id` claim correctly, and tier-slot model suggestions are aligned with the provider allowlist.
- **Post-4.6 Anthropic models registered.** The newer Anthropic lineup is now in the model registry with correct output ceilings, so a long write no longer gets truncated at an 8192-token cap.
- **Temperature omitted for Opus 4.7+ and Fable/Mythos.** These models reject any explicit temperature; the Anthropic and Bedrock paths now omit it, closing a 400 that appeared on the connection test and on normal turns.
- **Manual model id for Codex and custom providers.** Tier slots for ChatGPT OAuth and for custom providers accept a typed-in model id, so a newer model that the discovery call does not list yet can still be selected.
- **Cohere discoverability preset** added for OpenRouter-style setups.
- **Routing hardening.** A pinned chat model now reliably suppresses the TaskRouter; the tier regexes are safer (ambiguous GLM variants stay unclassified rather than mis-tiered); and the per-model "unclassified" log flood on startup is silenced.
- **CacheStat aggregation.** Sub-minimum cache-stat lines are aggregated instead of flooding the console one entry per call.
- **Self-development install hint.** A locally built plugin whose hash does not match the published asset now points you to "Install from file" instead of failing silently.

## Settings

- **Optional lean system prompt.** A new setting trims the system prompt to a leaner core for setups that want a smaller fixed-token footprint.

## Testing

The branch carries the full suite green (2666 passing, one intentional expected-fail), `tsc --noEmit` clean, ESLint clean on the changed files, production build clean (`main.js` 4.7 MB). The effort and retrieval work was developed test-first; the reasoning-effort and Bedrock-thinking changes were reviewed through adversarial multi-agent passes before merge. The Bedrock reasoning paths (effort enum and budget-token thinking) cannot be exercised against a live endpoint in CI and were verified by hand.
